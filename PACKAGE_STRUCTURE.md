# Package Structure

This document describes the internal architecture of `outputllsp3` and the
relationships between its modules.

---

## High-Level Architecture

```
Python source
     │
     │  (three entry paths)
     │
     ├─ Build script ──────────────────► transpiler.py
     │    build(project, api, ns)           │
     │                                      │
     ├─ Python-first ──────────────────► pythonfirst.py
     │    @robot.proc / @run.main            │
     │                                      │
     └─ AST (SPIKE Python) ────────────► ast_transpiler.py
          runloop.run(main())                │
                                            ▼
                                       project.py ◄── api.py / flow.py
                                       LLSP3Project        facades
                                            │
                                            ▼
                                       .llsp3 file
                                            │
                                       exporter.py
                                            │
                                            ▼
                                       Python source
                                       (round-trip)
```

## Module Groups

### Infrastructure

| Module | Role |
|--------|------|
| `version.py` | `__version__` constant — single source of truth |
| `metadata.py` | Package identity, `PACKAGE_LAYOUT`, `FEATURES`, `CHANGELOG`, `package_info()` |
| `enums.py` | `StrEnum` subclasses for hardware constants (`MotorPair`, `Port`, `Button`, …); `ENUMS` namespace |

### Core

| Module | Role |
|--------|------|
| `parser.py` | `parse_llsp3(path)` — reads `.llsp3` / `.llsp` archives; `LLSP3Document` dataclass |
| `project.py` | `LLSP3Project` — in-memory block graph; block creation, variables, procedures, save |
| `catalog.py` | `BlockCatalog` — block template registry built from `strings.json` |
| `schema.py` | `SchemaRegistry` / `bundled_schema()` — verified-opcode registry |

### Authoring Facades

| Module | Role |
|--------|------|
| `api.py` | `API` aggregate + sub-facades: `VarsAPI`, `ListsAPI`, `OpsAPI`, `WaitAPI`, `SensorAPI`, `MotorAPI`, `MoveAPI`, `DrivebaseAPI`, `RobotAPI` |
| `flow.py` | `FlowBuilder` — control-flow helpers: `start`, `procedure`, `call`, `if_`, `repeat`, `chain`, `seq`, `comment` |
| `wrapper.py` | `ScratchWrapper` — module-discovery facade; enumerate/describe/invoke blocks by opcode |
| `spikepython.py` | `SpikePythonAPI` — SPIKE Python library facade (`hub`, `motor`, `motor_pair`) |

### Transpilers

| Module | Role |
|--------|------|
| `transpiler.py` | Classic build-script loader: `transpile_path`, `transpile_file`, `transpile_package`, `transpile_module`, `autodiscover` |
| `ast_transpiler.py` | Python AST → Scratch blocks: `ASTBuilder`, `transpile_python_source` |
| `pythonfirst.py` | Decorator-based transpiler: `PythonFirstContext`, `robot`, `run`, `port`, `ls`, `transpile_pythonfirst_file` |

### Export

| Module | Role |
|--------|------|
| `exporter.py` | `.llsp3` → Python decompiler: `export_llsp3_to_python` with `raw`, `builder`, and `python-first` styles |

### Workflow

| Module | Role |
|--------|------|
| `workflow.py` | `bundled_paths`, `discover_defaults`, `doctor_report`, `init_workspace`, `roundtrip_llsp3`, `docs_index` |
| `cli.py` | `main()` — `argparse`-based CLI entry-point for all sub-commands |

### Resources (bundled data)

| File | Contents |
|------|----------|
| `resources/ok.llsp3` | Minimal LLSP3 template used as base for generated projects |
| `resources/full.llsp3` | Full LLSP3 template with all blocks available |
| `resources/block_reference.llsp3` | Block-reference project used for opcode schema extraction |
| `resources/strings.json` | Scratch block string definitions (labels, menus, argument types) |

## Dependency Graph

Dependencies flow **downward** — no module imports from a module above it.

```
cli.py
  └─ workflow.py, transpiler.py, ast_transpiler.py, pythonfirst.py, exporter.py
       └─ api.py, flow.py
            └─ project.py, wrapper.py, spikepython.py
                 └─ catalog.py, schema.py, parser.py, enums.py
                      └─ metadata.py
                           └─ version.py
```

Circular imports are avoided by importing at function scope in `transpiler.py`
(the `from .pythonfirst import transpile_pythonfirst_file` at the bottom of
`transpiler.py` is intentional — it makes the function available via
`transpile_path` without creating a cycle at module-load time).

## Block Creation Model

Every block in a Scratch project has:

- A unique string **ID** (generated by `LLSP3Project._id()`)
- An **opcode** string (e.g. `"flippermotor_motorRunFor"`)
- **inputs** — child block references or literal values
- **fields** — enum/constant selections stored directly in the block
- **parent** / **next** — doubly-linked list pointers for sequencing
- **topLevel** / **x** / **y** — canvas position (for hat blocks only)

`LLSP3Project.add_block()` handles ID generation, parent-pointer repair,
input wiring, and block registration in `self.blocks`.

The facades in `api.py` and `flow.py` call `add_block()` indirectly through
named helpers like `project.set_variable()`, `project.repeat()`, etc.

---

## Custom Procedure Compilation

Scratch custom procedures use three block types:

1. **`procedures_prototype`** — declares the signature (proccode, argumentids,
   argumentnames, argumentdefaults)
2. **`procedures_definition`** — hat block that holds the prototype and body
3. **`procedures_call`** — call site block with input slots for each argument

In python-first / AST mode the compiler:
1. Calls `project.define_procedure(name, args, defaults=…)` to create the
   definition + prototype
2. Appends the body blocks via `project.attach_procedure_body(name, …)`
3. Emits `project.call_procedure(name, args)` at each call site, with missing
   positional args filled from `_proc_meta[name]["defaults"]`

## Strict Verified Mode

When `project.set_strict_verified(True)` is called, every `add_block()` call
checks the opcode against the `SchemaRegistry` built from
`resources/block_reference.llsp3`.  An unknown opcode raises `ValueError`
immediately rather than producing a project that the SPIKE app silently rejects.

Enable via CLI with `--strict-verified` on the `build` / `build-python`
sub-commands.
